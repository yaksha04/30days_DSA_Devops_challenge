Bucket Policies vs ACLs

Bucket Policy (Modern way — use this)

JSON-based (IAM-style)
Applied at bucket level
Supports conditions (IP, time, etc.)
Centralized control

ACL (Old + limited)

Simple read/write permissions
No conditions, no flexibility
Applied at object & bucket level

👉 Reality check:
If you’re still relying on ACLs in production, something is off. AWS itself recommends disabling ACLs (Bucket Owner Enforced) and using policies.

📦 S3 Versioning & MFA Delete

Versioning

Keeps multiple versions of an object
Protects from accidental deletion/overwrite

Example:

Upload file.txt → version v1
Upload again → version v2 (v1 still exists)

MFA Delete

Requires MFA to:
Delete object versions
Disable versioning

👉 Important:

Only root user can enable MFA delete (annoying but true)
Rarely used in startups, but valuable in high-security setups
🔄 S3 Lifecycle Rules & Transitions

Automates storage cost optimization.

Example flow:

Day 0 → Standard
Day 30 → Standard-IA
Day 90 → Glacier
Day 365 → Delete

👉 Used when:

Logs, backups, old data

👉 Pro tip:
If you're not using lifecycle rules, you're basically burning money 🔥

🔗 Pre-signed URLs

Temporary access to private S3 objects.

Example:

Backend generates URL → valid for 5 minutes
User downloads without needing AWS credentials

👉 Use cases:

Secure file sharing
Upload/download from frontend directly
🚀 S3 Transfer Acceleration

Faster uploads/downloads using AWS edge locations.

How:

Uses Amazon CloudFront edge network

👉 When useful:

Users far from S3 region (global apps)

👉 When NOT useful:

Same-region uploads → waste of money
🌐 Static Website Hosting on S3

You can host:

HTML, CSS, JS

Steps:

Enable static hosting
Upload files
Set index + error document
Make bucket public (carefully!)

👉 Limitations:

No backend
No authentication logic

👉 Real-world:
Usually paired with:

Amazon CloudFront (CDN)
Route 53 (domain)
🔔 S3 Event Notifications

Trigger actions when events happen:

Upload
Delete
Restore

Destinations:

AWS Lambda
Amazon SNS
Amazon SQS

👉 Example:
Upload image → Lambda resizes → saves thumbnail

🌍 Cross-Region Replication (CRR)

Automatically copies objects to another region.

👉 Why:

Disaster recovery
Compliance
Low latency access

👉 Requirements:

Versioning must be enabled
IAM role required

👉 Reality:
CRR = extra cost
So only use if you actually need multi-region durability

🧠 S3 Intelligent-Tiering

Automatically moves data between tiers based on access.

Tiers:

Frequent
Infrequent
Archive

👉 Best for:

Unknown access patterns

👉 Cost:

Small monitoring fee per object

👉 Honest truth:
If your access pattern is predictable → lifecycle rules are cheaper
If not → intelligent tiering saves your brain

🧠 Final DevOps Insight

If you combine these smartly, you get a production-grade architecture:

Versioning + Lifecycle → cost + safety
Pre-signed URLs → secure access
Event Notifications → automation
CRR → disaster recovery
CloudFront + S3 → scalable frontend
